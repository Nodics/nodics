# Module Template

This directory is the scaffold template for a Nodics module. It shows the standard module layout for configuration, schemas, routers, services, pipelines, tests, and metadata.

Keep this template generic and capability-neutral. Generated modules should fill in ownership, extension points, contracts, and test expectations in their own README instead of inheriting vague placeholder text.

When framework conventions change, update this template so new modules start aligned with the same contract followed by human developers and AI tools.
